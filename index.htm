tipos de teste:

Teste de caixa preta:

Teste de caixa-preta é um teste de software para verificar a saída dos dados usando entradas de vários tipos. Tais entradas não são escolhidas conforme a estrutura do programa. Quanto mais entradas são fornecidas, mais rico será o teste.

Teste de Caixa branca

este de caixa-branca é uma técnica de teste que usa a perspectiva interna do sistema para modelar os casos de teste. No teste de software, a perspectiva interna significa basicamente o código fonte. No teste de hardware, cada nó de um circuito pode ser testado.

Teste de tarefa

é uma avaliação em que os participantes são solicitados a realizar uma série de atividades ou tarefas específicas relacionadas a uma habilidade, conhecimento ou competência em particular. Esse tipo de teste é comumente usado em diversos contextos, como recrutamento e seleção de pessoal, avaliação educacional, avaliação de habilidades técnicas, entre outros.

Teste comportamental

Um teste comportamental em programação, muitas vezes referido como teste de comportamento ou teste de aceitação, é uma prática de teste de software que se concentra em verificar se o software atende aos requisitos de comportamento especificados. Ele se concentra em validar o comportamento externo do sistema em vez de sua implementação interna

Testes de Interface

Teste de Interface é definido como um tipo de teste de software que verifica se a comunicação entre dois sistemas de software diferentes é feita corretamente. Uma conexão que integra dois componentes é chamada de interface. Essa interface no mundo da informática pode ser qualquer coisa como APIs, serviços da web, etc.

Teste de sistema

O teste do sistema é um tipo de teste de software que realiza verificações no sistema como um todo. Implica a integração de todos os módulos e componentes individuais do software que desenvolveu, para testar se o sistema funciona em conjunto como esperado.
